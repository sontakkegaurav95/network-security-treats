## ⚠️ Common Network Security Threats

### 1. Denial of Service (DoS) Attack

#### 🔹 What is DoS?
A DoS attack aims to make a server or network unavailable by overwhelming it with fake  excessive bot traffic.

#### 🔹 How it Works:
- Attacker sends a large number of fake requests to a server.
- Server becomes overloaded.
- fake requests make server unavailable
- Legitimate users cannot access the service.

#### 🔹 Impact:
- Server down
- Loss of revenue
- Poor user experience
- legitimate users face unavailability of server

#### 🔹 Real-World Example:
- GitHub faced a massive DDoS attack in 2018, causing temporary outage.

#### 🔹 Mitigation:
- Use firewalls and intrusion detection systems (IDS)
- Rate limiting
- Load balancing
- Cloud-based DDoS protection

---

### 2. Man-in-the-Middle (MITM) Attack

#### 🔹 What is MITM?
In this attack, the attacker secretly intercepts communication between two parties and listen the channels to steal the data or login creditials .

#### 🔹 How it Works:
- Attacker positions themselves between sender and receiver.
- Intercepts and possibly modifies communication.
- Steals the sensitive information .

#### 🔹 Impact:
- Data theft (passwords, credit card details)
- Privacy breach
- Hacker can gain Unauthorized access

#### 🔹 Real-World Example:
- Public Wi-Fi attacks where hackers intercept user data.

#### 🔹 Mitigation:
- Use HTTPS (SSL/TLS encryption)
- Avoid public Wi-Fi for sensitive transactions
- Use VPNs while visiting unsecured sites .
- Enable multi-factor authentication (MFA)

---

### 3. Spoofing Attack

#### 🔹 What is Spoofing?
Spoofing is when an attacker pretends to be a trusted user to gain unauthorized access.

#### 🔹 Types:
- IP Spoofing
- Email Spoofing
- DNS Spoofing

#### 🔹 How it Works:
- Fake identity is created
- Victim trusts the attacker
- Sensitive data is stolen

#### 🔹 Impact:
- Phishing attacks
- Data breaches
- Malware distribution

#### 🔹 Real-World Example:
- Fake emails pretending to be from banks asking for login details.

#### 🔹 Mitigation:
- Email filtering systems
- Verify sender identity
- Use secure DNS
- Anti-virus software

---

## 🛡️ Preventive Measures

- Install firewalls and antivirus software
- Keep systems updated
- Use strong passwords
- Enable encryption (HTTPS, VPN)
- Regular security audits
- Educate users about cyber threats

---

## 📊 Conclusion
Network security threats like DoS, MITM, and Spoofing can cause serious damage to systems and users. Understanding how these attacks work and implementing proper security measures can significantly reduce risks. Organizations and individuals must stay aware and proactive in securing their networks.
